บทเรียนที่ 10: การสร้างระบบเครือข่ายและความปลอดภัยส่วนบุคคล (PiVPN, Tailscale และ Pi-hole)

ในบทเรียนเสริมนี้ เราจะศึกษาทฤษฎีและแนวทางปฏิบัติในการจัดทำช่องทางการเข้าถึงเซิร์ฟเวอร์หลังบ้านอย่างปลอดภัย (Private VPN) ควบคู่กับระบบควบคุมชื่อโดเมนและตัวกรองโฆษณาในระดับเครือข่าย (DNS Sinkhole) เพื่อบูรณาการระบบ IoT Hub ให้มีความปลอดภัยและความสะดวกสูงสุดในการใช้งานจริง

Part 1: ทฤษฎีแนวคิดเครือข่าย และการวิเคราะห์ทรัพยากรระบบ

1. ความเข้าใจด้านเทคโนโลยี VPN (Virtual Private Network)

การเชื่อมต่อเข้าหาเซิร์ฟเวอร์ IoT Hub นอกบ้านโดยทั่วไปมีสองรูปแบบหลัก:

  • Client-Server VPN (เช่น PiVPN): อุปกรณ์ปลายทาง (Client) จะวิ่งเชื่อมต่อเข้าหาเกตเวย์ตัวกลาง (Server) ที่ตั้งอยู่ที่บ้านโดยตรง รูปแบบนี้จำเป็นต้องทำ Port Forwarding บนเราเตอร์บ้านเพื่อเปิดประตูให้ทราฟฟิกภายนอกเข้ามาได้
  • Peer-to-Peer Mesh VPN (เช่น Tailscale): อุปกรณ์ทุกตัวเชื่อมต่อถึงกันเป็นใยแมงมุม (Mesh Network) โดยมีตัวกลางคอยช่วยทำ NAT Traversal ข้ามผ่านไฟร์วอลล์ของเราเตอร์โดยตรง ทำให้ไม่ต้องเปิดพอร์ตใดๆ บนเราเตอร์

2. การทำงานของ DNS Sinkhole (Pi-hole)

เมื่ออุปกรณ์ใดๆ พยายามเข้าถึงเว็บไซต์ เครื่องจะส่งคำถามไปที่ DNS Server ก่อนเสมอ Pi-hole จะคอยทำหน้าที่เป็นผู้ตอบคำถาม DNS โดยเปรียบเทียบชื่อโฮสต์กับบัญชีดำ (Blocklists) หากพบว่าเป็นโฮสต์โฆษณาหรือตัวติดตาม Pi-hole จะส่งค่าไอพีปลายทางที่ไม่มีอยู่จริง (0.0.0.0 หรือ Sinkhole) กลับไป ทำให้เครื่อง Client ไม่สามารถดาวน์โหลดโฆษณาเหล่านั้นได้ ช่วยประหยัดแบนด์วิธและป้องกันภัยคุกคามในระดับเครือข่าย

⚠️ คำแจ้งเตือนและการวิเคราะห์ทรัพยากร (Resource Constraints):

บอร์ด Raspberry Pi Zero 2 W มีหน่วยความจำจำกัดเพียง 512MB RAM การเปิดรันโมดูลต่างๆ พร้อมกันทั้งหมดในระบบ (Mosquitto + FastAPI + SQLite + Tailscale + Pi-hole + PiVPN) อาจเกิดปัญหา Out of Memory (OOM) จนเครื่องค้างได้ง่าย สถิติการกินทรัพยากรปกติมีดังนี้:

  • Mosquitto Broker: ~10MB RAM
  • FastAPI & SQLite (WAL Mode): ~50MB - 80MB RAM
  • Tailscale Daemon: ~30MB - 50MB RAM
  • WireGuard VPN (Kernel Space): ~15MB RAM (กินแรมและ CPU ต่ำมาก)
  • OpenVPN (User Space): ~25MB - 40MB RAM (กิน CPU มากกว่า)
  • Pi-hole (FTL Engine + Admin Web): ~60MB - 120MB+ RAM (ตามปริมาณคิวรีและขนาด Ad-lists)

คำแนะนำระดับปฏิบัติ (Best Practices): หากต้องการเปิดรันบริการเครือข่ายขนาดใหญ่เหล่านี้ทั้งหมดร่วมกัน แนะนำให้เปลี่ยนมาใช้งานบอร์ดที่มีแรมสูงขึ้น เช่น Raspberry Pi 4 (2GB/4GB/8GB) หรือ Raspberry Pi 5 (4GB/8GB) หรือติดตั้งลงบนระบบปฏิบัติการ Ubuntu Desktop/Server บนเครื่องคอมพิวเตอร์ทั่วไปได้โดยไม่มีข้อจำกัด

Part 2: การติดตั้งแบบติดตั้งตรงบนระบบปฏิบัติการ (Bare-metal Installation)

แนวทางการติดตั้งตรงบนระบบปฏิบัติการหลัก (Host OS) ช่วยลดความซับซ้อนเรื่องสิทธิ์เครือข่าย และช่วยให้เครื่องมือแต่ละตัวทำงานสอดประสานกันได้อย่างเป็นธรรมชาติตามมาตรฐาน

1. ขั้นตอนการติดตั้ง Pi-hole (Bare-metal)

รันคำสั่งสคริปต์อัตโนมัติของโครงการทางการ:

curl -sSL https://install.pi-hole.net | bash

ทำตามขั้นตอนบนหน้าจอสีน้ำเงินเพื่อตั้งค่า Static IP และอินเตอร์เฟสหลัก เช่น eth0 (หากใช้สาย LAN) หรือ wlan0 (หากใช้ Wi-Fi) เมื่อเสร็จสิ้นระบบจะแสดงรหัสผ่านผู้ดูแลสำหรับล็อกอินทางหน้าเว็บควบคุม

2. ขั้นตอนการติดตั้ง PiVPN (Bare-metal)

รันตัวติดตั้งของ PiVPN ซึ่งจะช่วยจัดตั้งเซิร์ฟเวอร์ WireGuard หรือ OpenVPN ให้อย่างรวดเร็ว:

curl -L https://install.pivpn.io | bash

ข้อแนะนำการจูนความปลอดภัยและการบูรณาการ DNS:

  1. ในขั้นตอนเลือกโปรโตคอล แนะนำให้เลือก WireGuard เนื่องจากทำงานได้เร็วกว่าและประหยัดทรัพยากรของบอร์ดอย่างมาก
  2. ระหว่างการติดตั้ง ระบบจะตรวจสอบพบว่ามีบริการ Pi-hole รันอยู่บนเครื่อง และจะถามว่าต้องการใช้ Pi-hole เป็นตัวบริการ DNS หลักให้กับ Client ที่ต่อเข้ามาหรือไม่ ให้ตอบตกลง (Yes)
  3. ตั้งค่าการส่งผ่านพอร์ต (Port Forwarding) บนเราเตอร์ของคุณให้ตรงกับพอร์ตของ WireGuard (พอร์ตมาตรฐานคือ UDP 51820) ชี้มาที่ IP ของบอร์ด Raspberry Pi

3. การจัดการสิทธิ์และการเชื่อมต่อ Client

ใช้คำสั่งใน Terminal ในการเพิ่มโปรไฟล์ผู้ใช้และเชื่อมต่อผ่านสมาร์ทโฟน:

# สร้างโปรไฟล์สำหรับผู้ใช้งานใหม่
pivpn add

# แสดงรหัส QR Code บนหน้าจอ เพื่อใช้แอป WireGuard บนมือถือสแกนเชื่อมต่อ
pivpn -qr

ด้วยสถาปัตยกรรมนี้ โทรศัพท์มือถือที่อยู่นอกบ้านจะเสมือนนั่งอยู่ในบ้าน ได้รับการบล็อกโฆษณาจาก Pi-hole และสามารถพิมพ์เข้าหาหน้าแดชบอร์ดด้วยชื่อโดเมนภายในบ้าน (เช่น iothub.local) ได้โดยตรง

Part 3: ทางเลือกการรัน Pi-hole บน Docker และการแก้ปัญหาพอร์ต 53 ชน

กรณีที่ต้องการเก็บบอร์ดของท่านให้สะอาด และรันทุกอย่างภายใต้ Docker Compose เราก็สามารถคอนเทนเนอร์ไรซ์ Pi-hole ได้เช่นเดียวกัน

1. โครงสร้าง Docker Compose สำหรับ Pi-hole

version: "3"
services:
  pihole:
    container_name: pihole
    image: pihole/pihole:latest
    ports:
      - "53:53/tcp"
      - "53:53/udp"
      - "80:80/tcp"  # พอร์ตแอดมินเพจ (อาจเปลี่ยนเป็น 8080 หาก 80 ชนกับ FastAPI)
    environment:
      TZ: 'Asia/Bangkok'
      WEBPASSWORD: 'your_secure_password'
    volumes:
      - './etc-pihole:/etc/pihole'
      - './etc-dnsmasq.d:/etc/dnsmasq.d'
    cap_add:
      - NET_ADMIN  # จำเป็นอย่างยิ่งสำหรับการจัดการสิทธิ์ควบคุมเส้นทางเครือข่าย
    restart: unless-stopped
🔧 การแก้ปัญหาพอร์ต 53 ชน (systemd-resolved Conflict):

ในระบบปฏิบัติการ Ubuntu หรือ Raspberry Pi OS รุ่นใหม่ บริการเริ่มต้นของระบบชื่อ systemd-resolved จะรันและถือสิทธิ์พอร์ต 53 (Local DNS resolver) ไว้เสมอ ส่งผลให้การสั่งสตาร์ท Docker Pi-hole ล้มเหลว วิธีปิดกั้นและแก้ไขมีดังนี้:

  1. เปิดไฟล์คอนฟิกเพื่อแก้ไข: sudo nano /etc/systemd/resolved.conf
  2. แก้ไขหรือเพิ่มบรรทัดดังต่อไปนี้เพื่อปิดการทำงานในระดับ Stub: 
    DNSStubListener=no
  3. สร้างลิงก์ไฟล์ DNS ชี้ไปที่เนมเซิร์ฟเวอร์ระบบโดยตรงเพื่อป้องกันเน็ตหลุด: 
    sudo ln -sf /run/systemd/resolve/resolv.conf /etc/resolv.conf
  4. สั่งรีสตาร์ทบริการเพื่อใช้การตั้งค่าใหม่: sudo systemctl restart systemd-resolved
  5. หลังจากนี้จะสามารถรันคำสั่ง docker compose up -d เพื่อเปิดบริการ Pi-hole ได้โดยไม่พบข้อผิดพลาดเรื่อง Bind Port 53 ชนกัน

Part 4: การเข้าถึงระยะไกลด้วย Tailscale ร่วมกับ Pi-hole (Tailnet Integration)

หากคุณไม่ต้องการเปิดพอร์ตเราเตอร์บ้านเลยแม้แต่พอร์ตเดียว (ป้องกันการสแกนภายนอก 100%) การใช้ Tailscale P2P VPN ควบคู่กับ Pi-hole คือคำตอบที่เป็นมาตรฐานสูงสุด

1. ขั้นตอนปฏิบัติบนอุปกรณ์

ติดตั้งและเชื่อมต่อบอร์ด Raspberry Pi เข้าสู่ Tailnet ส่วนตัว:

curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

ระบบจะสร้าง IP ประจำตัวให้บอร์ดในวงเครือข่าย Tailscale (เช่น 100.115.x.y)

2. การตั้งค่าผ่านหน้าเว็บควบคุมของ Tailscale (Admin Console)

เพื่อให้ทุกอุปกรณ์ที่ติดตั้ง Tailscale ค้นหา DNS และบล็อกโฆษณาผ่าน Pi-hole อัตโนมัติ:

  1. เข้าไปที่หน้าแดชบอร์ดบริการ Tailscale Admin Console
  2. ในแถบเมนู DNS เลื่อนลงไปที่ Nameservers กดเลือก Add nameserver แล้วคลิก Custom...
  3. ใส่ที่อยู่ IP ของเครื่อง Raspberry Pi ที่ได้มาจากระบบ Tailscale (ไอพีในวง 100.x.y.z) แล้วกดเซฟ
  4. เปิดปุ่มสวิตช์ Override local DNS เพื่อบังคับใช้ทั่วทั้งเครือข่าย
🔒 ความมั่นคงปลอดภัยระดับสูง: วิธีนี้จะทำให้ทราฟฟิกข้อมูล DNS ของเราถูกเข้ารหัสและเชื่อมตรงเข้าหา Pi-hole จากทุกที่ในโลกได้อย่างสมบูรณ์ โดยไม่ต้องทำการเปิดพอร์ต 53 หรือพอร์ตใดๆ ออกสู่เครือข่ายอินเทอร์เน็ตสาธารณะเลย

🧠 ตรวจสอบความเข้าใจ (Quick Quiz)

หากนำเซิร์ฟเวอร์ VPN (PiVPN) และตัวกรอง DNS (Pi-hole) มาติดตั้งใช้งานบนบอร์ดที่มี RAM เพียง 512MB เช่น Raspberry Pi Zero 2 W ร่วมกับ FastAPI และระบบหลักอื่นๆ จะส่งผลกระทบและข้อจำกัดอย่างไร?

A) ระบบจะปฏิเสธการเชื่อมต่อเครือข่ายและใช้งานไม่ได้ทันทีตั้งแต่ระดับบอร์ดแวร์
B) ตัวเครื่องอาจเจอปัญหาแรมเต็ม (Out of Memory) และทำงานค้างได้ง่าย ควรแก้ไขโดยปิดระบบ Logging หรือเปลี่ยนไปใช้บอร์ดรุ่นสูงกว่าที่มี RAM 2GB ขึ้นไป หรือใช้ Ubuntu PC
