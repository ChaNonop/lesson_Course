บทเรียนที่ 8: พื้นฐาน OOP และหลักการ 4 เสาหลัก (C++ & Python)

ในบทเรียนนี้ เราจะปูพื้นฐานการเขียนโปรแกรมเชิงวัตถุ (Object-Oriented Programming) ซึ่งเป็นแนวคิดหลักในการพัฒนาทั้งระบบหลังบ้านด้วย Python (FastAPI) และเฟิร์มแวร์อุปกรณ์ด้วย C++ (PlatformIO)

Part 1: คลาส ออบเจกต์ แอตทริบิวต์ และเมธอด (Class, Object, Attribute, Method)

OOP (Object-Oriented Programming) คือการเขียนโปรแกรมที่มองทุกสิ่งเป็น "วัตถุ" (Object) ซึ่งถูกสร้างขึ้นจาก "แม่แบบ" หรือ "พิมพ์เขียว" (Class) โดยวัตถุแต่ละชิ้นจะประกอบไปด้วย:

  • Attribute (คุณลักษณะ/ข้อมูล): ตัวแปรที่เก็บสถานะของวัตถุนั้น เช่น ชื่อเซ็นเซอร์, ขาพอร์ตต่อเชื่อม, ค่าที่อ่านได้ล่าสุด
  • Method (พฤติกรรม/ฟังก์ชัน): ฟังก์ชันภายในคลาสที่สั่งการวัตถุให้ทำสิ่งต่างๆ เช่น สั่งให้อ่านค่าจากพิน, สั่งเชื่อมต่อเน็ตเวิร์ก

เปรียบเทียบโครงสร้างใน Python และ C++

ฝั่ง Python (Backend):

class Sensor:
    # Constructor (__init__) สำหรับกำหนดค่าเริ่มต้น
    def __init__(self, name: str, pin: int):
        self.name = name  # Attribute
        self.pin = pin    # Attribute

    # Method
    def read_value(self) -> float:
        print(f"Reading from {self.name} on pin {self.pin}")
        return 25.5

# การสร้างออบเจกต์ (Instance)
temp_sensor = Sensor("DHT11", 4)
val = temp_sensor.read_value()

ฝั่ง C++ (Firmware):

#include <Arduino.h>

class Sensor {
public:
    String name;  // Attribute
    int pin;      // Attribute

    // Constructor (ชื่อเดียวกับ Class)
    Sensor(String n, int p) {
        name = n;
        pin = p;
    }

    // Method
    float readValue() {
        Serial.print("Reading from " + name);
        return 25.5;
    }
};

// การสร้างออบเจกต์และเรียกใช้งาน Method (เรียกใช้งานภายในฟังก์ชันของ Arduino)
void setup() {
    Serial.begin(115200);
    Sensor tempSensor("DHT11", 4);
    float val = tempSensor.readValue();
}

void loop() {}

Part 2: การห่อหุ้มข้อมูล (Encapsulation)

Encapsulation (การห่อหุ้ม): คือการจำกัดการเข้าถึงข้อมูลภายในออบเจกต์จากโค้ดภายนอก เพื่อป้องกันการแก้ไขค่าตัวแปรโดยตรงโดยไม่ผ่านฟังก์ชันการตรวจสอบความสะอาด (Getter/Setter) ช่วยรักษาเสถียรภาพและระบบความปลอดภัย

1. การทำ Encapsulation ใน C++

ใช้คีย์เวิร์ดควบคุมการเข้าถึง (Access Specifiers) ได้แก่:

  • public: โค้ดภายนอกสามารถเรียกใช้และแก้ไขข้อมูลได้โดยตรง
  • private: จำกัดให้เฉพาะฟังก์ชันที่อยู่ในคลาสเดียวกันเท่านั้นที่เข้าถึงได้
class SecureMqtt {
private:
    String password; // ซ่อนไม่ให้ภายนอกเห็นหรือแก้ไขได้โดยตรง

public:
    // Getter function
    String getPassword() {
        return password;
    }

    // Setter function พร้อมการตรวจสอบความปลอดภัยก่อนเขียนทับ
    void setPassword(String pass) {
        if (pass.length() >= 8) { // ป้องกันการตั้งรหัสผ่านสั้นเกินไป
            password = pass;
        }
    }
};

2. การทำ Encapsulation ใน Python

Python ไม่มีคีย์เวิร์ด `private` โดยตรง แต่ใช้ข้อตกลงในการใส่เครื่องหมายนำหน้าตัวแปร:

  • _variable (Single Underscore): เป็นข้อตกลงว่าตัวแปรนี้ใช้ภายในคลาสเท่านั้น (Protected) โค้ดด้านนอกไม่ควรแก้ไขตรงๆ
  • __variable (Double Underscore / Name Mangling): เป็นการสั่งให้ Python ซ่อนชื่อตัวแปรนี้ ป้องกันการแก้ไขจากภายนอกโดยตรง 100%
class DBConnection:
    def __init__(self):
        self.__db_url = "sqlite:///./secure.db"  # ซ่อนแอตทริบิวต์ (Private)

    # Getter
    def get_url(self):
        return self.__db_url

    # Setter พร้อมการตรวจสอบความสมบูรณ์
    def set_url(self, new_url: str):
        if new_url.startswith("sqlite:///"):
            self.__db_url = new_url

Part 3: การสืบทอดคุณลักษณะ (Inheritance)

Inheritance (การสืบทอด): คือการสร้างคลาสใหม่ (Child Class / Subclass) โดยสืบทอดตัวแปรและฟังก์ชันทั้งหมดมาจากคลาสที่มีอยู่แล้ว (Parent Class / Superclass) ช่วยลดความซ้ำซ้อนในการเขียนโค้ด (Code Reusability)

1. การสืบทอดใน Python

# Class แม่ (Parent)
class NetworkDevice:
    def __init__(self, ip: str):
        self.ip = ip

    def ping(self):
        return f"Ping to {self.ip} OK"

# Class ลูก (Child) สืบทอดคุณลักษณะและฟังก์ชันจาก NetworkDevice
class ESP8266Device(NetworkDevice):
    def __init__(self, ip: str, chip_id: str):
        super().__init__(ip)  # เรียก Constructor ของ Class แม่มาทำงาน
        self.chip_id = chip_id

    # สามารถสร้างฟังก์ชันเฉพาะตัวเพิ่มได้
    def get_chip_id(self):
        return self.chip_id

esp = ESP8266Device("192.168.1.50", "ESP8266_A420")
print(esp.ping()) # เรียกใช้ฟังก์ชัน ping() ของคลาสแม่ได้ทันที

2. การสืบทอดใน C++

// Class แม่
class BaseSensor {
public:
    int pin;
    BaseSensor(int p) { pin = p; }
    void init() { pinMode(pin, INPUT); }
};

// Class ลูก สืบทอดแบบ public จาก BaseSensor
class AnalogRainSensor : public BaseSensor {
public:
    // โทรหา Constructor คลาสแม่ด้วย : BaseSensor(p)
    AnalogRainSensor(int p) : BaseSensor(p) {}

    int readRain() {
        return analogRead(pin);
    }
};

Part 4: การพหุสัณฐานและการทำงานแบบนามธรรม (Polymorphism & Abstraction)

1. Polymorphism (การพหุสัณฐาน/หลายรูปแบบ)

คือความสามารถของออบเจกต์คลาสลูกต่าง ๆ ในการตอบสนองต่อคำสั่งแบบเดียวกันในรูปแบบเฉพาะตัวของมันเอง (การเขียนทับเมธอด หรือ Method Overriding):

ตัวอย่างใน C++ (ด้วย Virtual Functions):

class Device {
public:
    // กำหนดให้เป็น Virtual function เพื่อให้คลาสลูกไปเขียนทับได้
    virtual void turnOn() {
        Serial.println("Generic device power on");
    }
};

class LEDDevice : public Device {
public:
    void turnOn() override { // เขียนทับฟังก์ชันของคลาสแม่
        digitalWrite(2, HIGH);
        Serial.println("LED turned ON!");
    }
};

2. Abstraction (การซ่อนรายละเอียดของระบบ/การสร้างโครงร่าง)

คือการออกแบบโครงร่างหรือสัญญาข้อตกลง (Interface) โดยระบุว่าคลาสที่จะสืบทอดไปต้องมีฟังก์ชันใดบ้าง แต่ไม่ต้องเขียนรายละเอียดการทำงานจริงในคลาสแม่ ปล่อยให้คลาสลูกไปพัฒนาเอง:

ตัวอย่าง Abstract Class ใน Python (ใช้ abc module):

from abc import ABC, abstractmethod

# กำหนดเป็นคลาส Abstract (ไม่สามารถนำมารันสร้างออบเจกต์ตรงๆ ได้)
class MqttConnection(ABC):
    @abstractmethod
    def connect(self):
        pass

class MyMqttClient(MqttConnection):
    def connect(self):
        # พัฒนาโค้ดเชื่อมต่อจริงที่นี่
        print("Connected to Mosquitto MQTT Broker!")
