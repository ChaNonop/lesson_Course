บทเรียนที่ 4: การจัดทำระบบ FastAPI และ SQLite

ในบทเรียนนี้ เราจะเริ่มต้นควบคุมสภาพแวดล้อมไพทอนด้วย venv, เรียนรู้การทำงานร่วมกับ SQLite ด้วยโหมดประสิทธิภาพสูงสุด (WAL Mode), ตรวจสอบข้อมูลก่อนเข้าสู่ DB ด้วย Pydantic, และปิดท้ายด้วยการควบคุมบริการผ่าน Docker-Compose

Part 1: Python Virtual Environment (venv)

เพื่อหลีกเลี่ยงปัญหาเวอร์ชันของไลบรารีทับกันในระบบปฏิบัติการหลัก เราจะตั้งกล่องทรายแยกสำหรับการพัฒนาเว็บแอป:

1. ติดตั้ง venv (สำหรับบางระบบที่ยังไม่มี):

sudo apt install -y python3-venv python3-pip

2. สร้างกล่องทราย venv และเปิดใช้งาน:

python3 -m venv venv
source venv/bin/activate

เมื่อสังเกตเห็นคำว่า (venv) ขึ้นที่ด้านหน้าของ Terminal หมายความว่าเครื่องทำงานบนสภาพแวดล้อมจำลองเรียบร้อยแล้ว

Part 2: SQLite Syntax & Performance Tuning (WAL Mode)

SQLite เป็นฐานข้อมูลแบบไฟล์เดี่ยว มีปัญหาคลาสสิกคือเมื่อมีการอ่านข้อเขียน (Write) พร้อม ๆ กันจากหลายเอนทิตี (เช่น ESP32 กำลังเขียน และ FastAPI กำลังรับข้อมูลแล้วเขียน) จะขึ้นข้อผิดพลาด database is locked เราจะจูนระบบด้วย WAL Mode:

WAL (Write-Ahead Logging) Mode: ช่วยให้ผู้อ่าน (Reader) สามารถอ่านค่าจากฐานข้อมูลได้ในขณะที่ผู้เขียน (Writer) กำลังเขียนข้อมูลลงหน้ากระดาษจำลองใหม่ในไฟล์ .db-wal ชั่วคราว ป้องกัน Database Locked ได้ 100%

ตัวอย่างการเปิดใช้งาน WAL Mode ใน database.py ด้วย SQLAlchemy Engine:

from sqlalchemy import create_engine
from sqlalchemy.orm import declarative_base, sessionmaker

SQLALCHEMY_DATABASE_URL = "sqlite:///./sensor_data.db"

# 💡 1. ตั้งค่า timeout ใน connect_args (Busy Timeout) เพื่อรอคิวก่อนยกเลิกข้อผิดพลาด
# 💡 2. รันคำสั่ง PRAGMA journal_mode=WAL ทันทีที่เชื่อมต่อฐานข้อมูล
engine = create_engine(
    SQLALCHEMY_DATABASE_URL, 
    connect_args={"check_same_thread": False, "timeout": 30} 
)

@event.listens_for(engine, "connect")
def set_sqlite_pragma(dbapi_connection, connection_record):
    cursor = dbapi_connection.cursor()
    cursor.execute("PRAGMA journal_mode=WAL")
    cursor.execute("PRAGMA synchronous=NORMAL")
    cursor.close()

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

Part 2.5: Time-based SQL Queries (การเขียนคำสั่งดึงข้อมูลย้อนหลังตามเวลา)

เมื่อต้องการนำเสนอประวัติข้อมูลย้อนหลังตามเวลาในแดชบอร์ด (เช่น 1 นาที, 15 นาที, 1 วัน หรือ 1 สัปดาห์ก่อนหน้า) เราจะเขียนคำสั่ง SQL คัดกรองข้อมูลโดยใช้ฟังก์ชันจัดการเวลาของ SQLite ดังนี้:

-- 1. ดึงข้อมูลใน 1 นาทีล่าสุด
SELECT * FROM sensor_readings 
WHERE timestamp >= datetime('now', '-1 minute') 
ORDER BY timestamp DESC;

-- 2. ดึงข้อมูลใน 15 นาทีล่าสุด
SELECT * FROM sensor_readings 
WHERE timestamp >= datetime('now', '-15 minutes') 
ORDER BY timestamp DESC;

-- 3. ดึงข้อมูลใน 1 วันล่าสุด
SELECT * FROM sensor_readings 
WHERE timestamp >= datetime('now', '-1 day') 
ORDER BY timestamp DESC;

-- 4. ดึงข้อมูลใน 1 สัปดาห์ล่าสุด
SELECT * FROM sensor_readings 
WHERE timestamp >= datetime('now', '-7 days') 
ORDER BY timestamp DESC;

-- 5. ค้นหาค่าอุณหภูมิสูงสุดของวันนี้ (Max Temp of the Day)
SELECT MAX(value) FROM sensor_readings 
WHERE sensor_name = 'esp32_temp' 
AND timestamp >= datetime('now', 'start of day');

Part 3: ORM & Pydantic Data Validation

การใช้ Object Relational Mapping (ORM) ช่วยให้เราเขียนโปรแกรมไพทอนแทนการพิมพ์ดิบ SQL ควบคู่กับการควบคุมความสะอาดข้อมูลด้วย Pydantic:

1. models.py (SQLAlchemy ORM Model): กำหนดโครงสร้างตารางจริงในฐานข้อมูล

from sqlalchemy import Column, Integer, String, Float, DateTime
from datetime import datetime, timezone
from database import Base

class SensorReading(Base):
    __tablename__ = "sensor_readings"

    id = Column(Integer, primary_key=True, index=True)
    sensor_name = Column(String, index=True)
    value = Column(Float)
    timestamp = Column(DateTime, default=lambda: datetime.now(timezone.utc))

2. schemas.py (Pydantic Schema): ตรวจสอบว่าพารามิเตอร์ของ Request ถูกต้องตามชนิดข้อมูลและเงื่อนไขที่กำหนดหรือไม่ก่อนเซฟ

from pydantic import BaseModel, Field

class SensorReadingCreate(BaseModel):
    sensor_name: str = Field(..., min_length=3, max_length=50, example="esp32_temp")
    value: float = Field(..., ge=-50.0, le=100.0) # จำกัดอุณหภูมิให้อยู่ในกรอบที่เป็นไปได้

Part 4: Docker & Docker Compose Orchestration

เรามัดรวมเว็บเซิร์ฟเวอร์ FastAPI และ MQTT Broker (Mosquitto) ด้วย Docker Compose เพื่อความสะดวกในการสร้างระบบแบบ One-command deployment:

1. Dockerfile (สำหรับ backend/FastAPI):

FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]

2. docker-compose.yml:

version: '3.8'

services:
  mqtt:
    image: eclipse-mosquitto:latest
    container_name: mosquitto_broker
    ports:
      - "1883:1883"
    volumes:
      - ./mosquitto/config:/mosquitto/config

  backend:
    build: ./backend
    container_name: fastapi_backend
    ports:
      - "8000:8000"
    volumes:
      - ./backend:/app # เมาท์เพื่อให้แก้ไขโค้ดได้เรียลไทม์
    depends_on:
      - mqtt
