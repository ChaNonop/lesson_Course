บทเรียนที่ 2: การควบคุม Terminal, tmux และติดตั้ง MQTT Broker

ในบทเรียนนี้ เราจะเจาะลึกการควบคุมเซิร์ฟเวอร์แบบ Command Line (CLI), การรักษาสถานะเซสชันด้วย tmux และกระบวนการติดตั้งและล็อกระบบความปลอดภัยของ MQTT Broker (Mosquitto)

Part 1: Linux Terminal & Journalctl Monitoring

การเป็นผู้พัฒนา IoT Hub จำเป็นต้องคุ้นเคยกับคำสั่ง CLI พื้นฐานเพื่อสอดส่องและดูแลระบบปฏิบัติการ Debian/Ubuntu บน Raspberry Pi:

  • top หรือ htop: ใช้ตรวจสอบการทำงานของ CPU และดูปริมาณการใช้ RAM ที่เหลืออยู่บนบอร์ด
  • df -h: ตรวจสอบความจุและการใช้งานดิสก์ (SD Card) เพื่อตรวจจับกรณี log หรือฐานข้อมูลโตจนเต็มดิสก์
  • journalctl -u mosquitto.service -n 50 -f: แสดงบันทึกเหตุการณ์ (Logs) ล่าสุด 50 บรรทัด และกดติดตามแบบเรียลไทม์ (Follow) เพื่อส่องข้อผิดพลาดของการเชื่อมต่อ MQTT

Part 2: tmux Session & Pane Control (Basic Usage)

ปัญหาหลักเมื่อเชื่อมต่อ SSH เข้าไปยังบอร์ดคอมพิวเตอร์ระยะไกลคือ หากหน้าต่าง SSH หลุดหรือเน็ตเวิร์กที่บ้านดับ โปรแกรมหรืองานรันเซิร์ฟเวอร์ที่ทำทิ้งไว้จะตายไปทันที เราแก้ปัญหานี้ด้วย tmux (Terminal Multiplexer) ซึ่งเป็นเครื่องมือช่วยจำลองเซสชันหน้าจอดำรันค้างไว้เบื้องหลัง แม้เราจะตัดการเชื่อมต่อออกไป:

1. การติดตั้ง tmux:

sudo apt update && sudo apt install -y tmux

2. การจัดการ Session หลัก (การเข้า-ออก):

  • สร้างเซสชันใหม่: รันคำสั่ง tmux new -s iothub (สร้างและเชื่อมเข้าไปในห้องทำงานชื่อ iothub)
  • ถอยออกมา (Detach): กดปุ่ม Ctrl+B จากนั้นปล่อยนิ้วแล้วกดปุ่ม D เพื่อถอยกลับไปหน้าหลัก (โปรเซสข้างในจะยังรันต่อเนื่องเป็น Background)
  • เชื่อมกลับเข้าห้องเดิม (Attach): รันคำสั่ง tmux attach -t iothub

3. การจัดการช่องหน้าจอ (Split Panes):

ภายใน tmux เราสามารถซอยหน้าจอเป็นหลาย ๆ บล็อกย่อย เพื่อส่อง Log ฝั่งหนึ่ง และรันคำสั่งรันเซิร์ฟเวอร์อีกฝั่งหนึ่งได้พร้อมกัน:

  • แบ่งหน้าจอ บน-ล่าง (Split Horizontally): กดปุ่ม Ctrl+B จากนั้นปล่อยนิ้วแล้วกดปุ่ม " (อัญประกาศคู่)
  • แบ่งหน้าจอ ซ้าย-ขวา (Split Vertically): กดปุ่ม Ctrl+B จากนั้นปล่อยนิ้วแล้วกดปุ่ม % (เปอร์เซ็นต์)
  • สลับบล็อกการทำงาน (Navigate): กดปุ่ม Ctrl+B จากนั้นปล่อยนิ้วแล้วกด ปุ่มลูกศร (Arrow Keys) ชี้ไปทางช่องที่ต้องการย้ายเคอร์เซอร์
  • ออกจากบล็อกย่อย (Exit Pane): พิมพ์คำสั่ง exit หรือกดคีย์ลัด Ctrl+D ภายในช่องนั้นเพื่อลบช่องดังกล่าวทิ้ง
  • ออกจาก tmux ถาวร: ปิดช่องย่อย (Panes) ทั้งหมดทีละช่อง หรือพิมพ์คำสั่ง exit ในช่องสุดท้าย เซสชัน tmux จะถูกทำลายลงอย่างสมบูรณ์

Part 3: Mosquitto Installation (Apt vs Docker)

เราสามารถติดตั้ง MQTT Broker (Mosquitto) ได้ 2 วิธีหลัก ขึ้นอยู่กับกลยุทธ์การบริหารจัดการระบบ:

วิธีที่ 1: ติดตั้งผ่าน apt (Native install)

sudo apt update
sudo apt install -y mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

วิธีที่ 2: ติดตั้งผ่าน Docker Container (แยกส่วนระบบแอป)

ใช้คำสั่งสวมคอนเทนเนอร์เพื่อให้ง่ายต่อการจัดการและเคลื่อนย้ายไปรันบนเครื่องอื่นได้รวดเร็ว (สำหรับรายละเอียดคำสั่งและสถาปัตยกรรมของ Docker สามารถเรียนเพิ่มเติมได้ใน บทเรียนที่ 3: พื้นฐาน Docker):

docker run -d --name mosquitto -p 1883:1883 -p 9001:9001 -v /mosquitto/config:/mosquitto/config eclipse-mosquitto

Part 4: MQTT Security & Authentication

การเปิดพอร์ตแบบไม่มีการยืนยันตัวตน (allow_anonymous true) ในสภาพแวดล้อมใช้งานจริงมีความเสี่ยงสูงมากที่จะโดนบุกรุก เราจึงสร้างระบบความปลอดภัยผ่าน Username/Password:

1. สร้างไฟล์รหัสผ่านของระบบและเพิ่มชื่อผู้ใช้ esp_node:

sudo mosquitto_passwd -c /etc/mosquitto/passwd esp_node

(พิมพ์ตั้งรหัสผ่านของคุณ จากนั้นกดยืนยัน)

2. แก้ไขไฟล์คอนฟิกหลักเพื่อปิดการเชื่อมต่อภายนอกแบบไร้ตัวตน และชี้ไปยังไฟล์รหัสผ่าน:

sudo nano /etc/mosquitto/conf.d/local.conf

แก้ไขข้อมูลข้างในเป็น:

listener 1883
allow_anonymous false
password_file /etc/mosquitto/passwd

3. รีสตาร์ท Broker เพื่ออัปเดตสิทธิ์ความปลอดภัย:

sudo systemctl restart mosquitto

4. ทดสอบความถูกต้องด้วยคำสั่ง Subscribe พร้อมส่งสิทธิ์ยืนยันตัวตน:

mosquitto_sub -h localhost -t "sensors/#" -u "esp_node" -P "รหัสผ่านที่ตั้งไว้"
