# บันทึกการเรียนรู้ 0002: ปรับปรุงดีไซน์สู่ Hex-Editor Technical Dark Theme

*   **วันที่:** 2026-07-04
*   **สถานะ:** เสร็จสมบูรณ์

---

## บริบทและปัญหา (Context)

ผู้เรียนมีความประสงค์ในการปรับโฉมบทเรียนการสอนจากธีมสว่างพาสเทลเดิมให้เป็น **Hex-editor / Memory Inspector Dark Theme** (หรือ Dual-Accent Technical Dark Theme) เพื่อความสวยงาม เป็นมืออาชีพ และสอดคล้องกับโทนของนักพัฒนาซอฟต์แวร์สายดีบักเกอร์/ฝังตัว (embedded)

ความต้องการหลัก:
1.  **พื้นหลังสีเข้มโทน graphite-blue (`#0F1419`):** ให้อารมณ์ซอฟต์แวร์เครื่องมือวิเคราะห์หน่วยความจำ/ดีบักเกอร์
2.  **Motif ของ Address Ruler (`0x00`, `0x08`, `0x10`...):** เพื่อเลียนแบบการแสดงผลของ memory addresses
3.  **ระบบสีคู่ (Dual-Accent):**
    *   **สีส้ม (`#E8944A`):** สื่อถึงความหมาย "List / Scattered" (ข้อมูลที่กระจายตัว/ฝั่ง Client เช่น ไมโครคอนโทรลเลอร์ ESP32, อุปกรณ์ IoT, และ Message payloads จาก MQTT)
    *   **สีฟ้า (`#4FD1C5`):** สื่อถึงความหมาย "NumPy / Contiguous" (ข้อมูลแบบต่อเนื่อง/โครงสร้างฝั่ง Server เช่น ฐานข้อมูล SQLite และการจัดเก็บถาวร)
4.  **ฟอนต์คู่:** `JetBrains Mono` สำหรับ Labels/Eyebrows/Code และ `Inter`/`Noto Sans Thai` สำหรับเนื้อหาภาษาไทยเพื่อให้อ่านง่าย

---

## การตัดสินใจ (Decision)

1.  **สร้างชุดไฟล์สไตล์และสคริปต์ส่วนกลาง (`style.css`, `main.js`):** เพื่อลดความซ้ำซ้อนของโค้ดสไตล์ในบทเรียนทั้ง 5 บท และใบอ้างอิงย่อ (Cheat Sheet) ช่วยให้การบำรุงรักษาในอนาคตสะดวกขึ้นมาก
2.  **ประมวลผลการจัดสไตล์และการจำแนกด้วย JavaScript (`main.js`):**
    *   จัดแบ่งโครงสร้าง HTML ใหม่แบบไดนามิกโดยเอาเนื้อหาไปบรรจุในกริดที่มี Address Ruler วิ่งอยู่ด้านซ้าย เพื่อบอกลำดับ offset `0x00`, `0x08`, `0x10`... อัตโนมัติ
    *   ใช้ระบบคัดกรองเนื้อหาแบบ Semantic เพื่อวิเคราะห์การใช้คีย์เวิร์ดของแต่ละการ์ด หากหัวข้อเป็นเรื่อง MQTT/ESP32 จะปรับสไตล์ขอบให้เป็นสีส้ม (Scattered) หากเป็นเรื่อง Database/SQLite/FastAPI จะปรับให้เป็นสีฟ้า (Contiguous)
    *   ยกระดับระบบคำถามท้ายบท (Quiz) จากคำสั่ง `alert()` ของเบราว์เซอร์ที่ดูไม่เป็นมืออาชีพ ให้เป็นการตอบคำถามแบบ Interactive ภายในหน้าเว็บ เผยเฉลยพร้อมคำอธิบายแบบไดนามิกทันที

---

## ผลลัพธ์และข้อสังเกต (Consequences)

*   **ความสวยงามและสม่ำเสมอ:** ทุกหน้าเข้าสู่หน้าจอดีบักเกอร์สีเทาดำสุดล้ำทันสมัย ตัวหนังสือสีครีมตัดพาสเทลส้มและฟ้าอย่างชัดเจน
*   **โค้ดสะอาดขึ้น:** ขนาดไฟล์ HTML ของแต่ละบทเรียนลดลงกว่า 50% เนื่องจากย้ายโค้ด CSS และระบบจัดการ Event Quiz ไปไว้ในไฟล์กลาง
*   **ประสบการณ์ผู้เรียน (UX) ที่ดีขึ้น:** มีแถบนำทาง (Nav-Menu) ด้านบนของทุกหน้าเพื่อให้เปลี่ยนบทเรียนไปมาได้ง่าย และการตรวจข้อสอบทำได้เรียบเนียนขึ้นโดยไม่มี Alert เด้งขัดจังหวะการเรียนรู้
