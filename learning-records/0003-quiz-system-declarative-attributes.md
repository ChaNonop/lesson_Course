# บันทึกการเรียนรู้ 0003: รีโนเวทระบบ Quiz ให้เป็นแบบ Declarative Attributes (HTML)

*   **วันที่:** 2026-07-04
*   **สถานะ:** เสร็จสมบูรณ์

---

## บริบทและปัญหา (Context)

จากเดิมที่บทเรียนทั้งหมด (Lessons 1-5) ใช้งานตัวตรวจข้อสอบผ่านคำสั่ง inline event handler `onclick="alert(...)"` ของเบราว์เซอร์ ซึ่งถือเป็นวิธีที่ไม่เป็นระเบียบ (dirty markup) และจำเป็นต้องใช้ Regular Expression ในไฟล์สคริปต์หลัก (`main.js`) ในการดักจับข้อความใน `alert()` มาแปลงเป็นหน้าต่างแสดงผลแบบ inline

เพื่อความสะอาด ความมั่นคงปลอดภัย และง่ายต่อการปรับปรุงรูปแบบคำถามในอนาคต จึงจำเป็นต้องทำการรีโนเวทตัวเลือกตอบของข้อสอบแบบทดสอบ (Quiz options) ในไฟล์ HTML ทั้งหมด ให้ใช้รูปแบบของ Custom Data Attributes แทน

---

## การตัดสินใจ (Decision)

1.  **ปรับสไตล์โค้ด HTML ใหม่ (HTML Markup Refactoring):**
    *   นำคำสั่ง inline handler `onclick="alert(...)"` ออกจากทุกไฟล์ `.html`
    *   ใช้โครงสร้าง Custom Data Attributes แทน ได้แก่:
        *   `data-correct="true"` หรือ `data-correct="false"` เพื่อกำหนดว่าข้อนั้นถูกต้องหรือไม่
        *   `data-feedback="..."` เพื่อแสดงคำอธิบายหรือผลลัพธ์ที่จะโชว์ให้ผู้เรียนเห็นเมื่อคลิก
2.  **ปรับปรุง JavaScript ส่วนกลาง (`main.js`):**
    *   แก้ไขฟังก์ชัน `upgradeQuiz()` ให้อ่านค่าจาก `data-correct` และ `data-feedback` ได้โดยตรง ซึ่งมีเสถียรภาพมากกว่าและไม่ต้องใช้ regex ในการแกะตัวหนังสือ
    *   ยังคงมีระบบ Fallback ในกรณีที่บทเรียนเก่าบางตัวยังคงใช้สไตล์ `onclick="alert(...)"` เพื่อไม่ให้ระบบเดิมพัง (Backward Compatibility)

---

## ผลลัพธ์และข้อสังเกต (Consequences)

*   **โค้ด HTML สะอาดขึ้นอย่างมีนัยสำคัญ:** ปราศจากการเขียนคำสั่งสคริปต์ JavaScript ปนอยู่ในไฟล์ HTML markup
*   **บำรุงรักษาง่ายขึ้น:** ผู้พัฒนาหลักสูตรสามารถแก้ไขข้อความ Feedback หรือเปลี่ยนข้อที่ถูกต้องได้ผ่าน Attribute ตรงๆ โดยไม่ต้องพะวงเรื่อง syntax ของ JavaScript string escaping
*   **ระบบทำงานได้อย่างแม่นยำขึ้น:** การเช็กความถูกต้องผ่าน Boolean `data-correct="true"` มีเสถียรภาพกว่าการกวาด regex หาคำว่า "ถูกต้อง" ในประโยค
