📋 IoT Hub Quick Reference & Cheat Sheet

หน้านี้เป็นคู่มือสรุปคำสั่งตั้งค่าและโค้ดตัวอย่างแบบย่อ สำหรับนำไปเปิดดูย้อนหลังขณะใช้งานจริง เนื้อหาครอบคลุมทุกขั้นตอนตั้งแต่การควบคุม Terminal ไปจนถึงการเขียนเฟิร์มแวร์บอร์ด ESP32

1. คำสั่งควบคุม MQTT Broker (Mosquitto)

การติดตั้งผ่าน apt:

sudo apt update && sudo apt install -y mosquitto mosquitto-clients
sudo systemctl enable mosquitto
sudo systemctl start mosquitto

การตั้งค่าไฟล์คอนฟิก (/etc/mosquitto/conf.d/local.conf):

# แบบมี Authentication (แนะนำ)
listener 1883
allow_anonymous false
password_file /etc/mosquitto/passwd

# แบบไม่มี Auth (เฉพาะ LAN ที่ไว้วางใจ)
listener 1883
allow_anonymous true

การสร้างไฟล์รหัสผ่านและเพิ่มผู้ใช้:

# สร้างไฟล์ passwd และเพิ่มผู้ใช้ esp_node
sudo mosquitto_passwd -c /etc/mosquitto/passwd esp_node

# เพิ่มผู้ใช้เพิ่มเติม (ไม่ต้องใช้ -c อีก)
sudo mosquitto_passwd /etc/mosquitto/passwd backend_user

คำสั่งทดสอบ Pub/Sub พร้อม Authentication:

# รับข้อมูล (Subscribe)
mosquitto_sub -h localhost -t "sensors/#" -u "esp_node" -P "รหัสผ่าน"

# ส่งข้อมูล (Publish)
mosquitto_pub -h localhost -t "sensors/data" -m '{"sensor_name":"test","value":28.5}' -u "esp_node" -P "รหัสผ่าน"

คำสั่งจัดการ Service:

sudo systemctl start mosquitto
sudo systemctl stop mosquitto
sudo systemctl restart mosquitto
sudo systemctl status mosquitto

# ส่อง Log แบบเรียลไทม์
journalctl -u mosquitto.service -n 50 -f

2. การควบคุม tmux Session

คำสั่งพื้นฐาน:

# สร้าง Session ใหม่ชื่อ iothub
tmux new -s iothub

# รายการ Session ที่มีอยู่
tmux ls

# กลับเข้า Session ที่มีอยู่
tmux attach -t iothub

# ลบ Session ทิ้ง
tmux kill-session -t iothub

คีย์ลัดสำคัญ (กด Ctrl+B ก่อนเสมอ):

Ctrl+B → D           ถอยออก (Detach) — โปรเซสยังทำงานอยู่เบื้องหลัง
Ctrl+B → %           แบ่งหน้าจอซ้าย-ขวา (Split vertical)
Ctrl+B → "           แบ่งหน้าจอบน-ล่าง (Split horizontal)
Ctrl+B → ปุ่มลูกศร    สลับบล็อก/ช่องหน้าจอ (Navigate between panes)
exit หรือ Ctrl+D      ปิดหน้าต่างย่อย (Close active pane)
Ctrl+B → C           สร้างหน้าต่างใหม่ (New window)
Ctrl+B → [0-9]       สลับระหว่างหน้าต่างหลัก

3. Backend FastAPI & SQLAlchemy

ติดตั้ง Library (ภายใน venv):

python3 -m venv venv
source venv/bin/activate

pip install fastapi uvicorn sqlalchemy paho-mqtt aiofiles pydantic

รันเซิร์ฟเวอร์:

# รันแบบ Dev (--reload จะรีโหลดเมื่อแก้โค้ด)
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# เปิดดู Swagger API Docs
# http://[IP_Raspberry_Pi]:8000/docs

ตั้งค่า database.py พร้อม WAL Mode:

from sqlalchemy import create_engine, event
from sqlalchemy.orm import declarative_base, sessionmaker

SQLALCHEMY_DATABASE_URL = "sqlite:///./sensor_data.db"

engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    connect_args={"check_same_thread": False, "timeout": 30}
)

@event.listens_for(engine, "connect")
def set_sqlite_pragma(dbapi_connection, connection_record):
    cursor = dbapi_connection.cursor()
    cursor.execute("PRAGMA journal_mode=WAL")
    cursor.execute("PRAGMA synchronous=NORMAL")
    cursor.close()

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)
Base = declarative_base()

4. SQLite Database Commands

เปิดและสืบค้นข้อมูลจาก Terminal:

# เปิดไฟล์ฐานข้อมูล
sqlite3 sensor_data.db

# ดูตารางทั้งหมด
.tables

# ดูโครงสร้างตาราง
.schema sensor_readings

# ดึงข้อมูล 10 รายการล่าสุด
SELECT * FROM sensor_readings ORDER BY id DESC LIMIT 10;

# ออกจาก SQLite
.exit

ตรวจสอบสถานะ WAL Mode:

PRAGMA journal_mode;
-- ควรแสดงผล: wal

5. Docker & Docker Compose

คำสั่งควบคุม Docker Compose:

# สร้างและรัน Container ทั้งหมดเบื้องหลัง
docker compose up -d

# Build ใหม่หลังแก้โค้ด
docker compose up -d --build

# ส่อง Log แบบรวม
docker compose logs -f

# ส่อง Log เฉพาะ Service
docker compose logs -f backend

# ปิดและทำลาย Container
docker compose down

โครงสร้าง docker-compose.yml:

version: '3.8'
services:
  mqtt:
    image: eclipse-mosquitto:latest
    ports:
      - "1883:1883"
    volumes:
      - ./mosquitto/config:/mosquitto/config

  backend:
    build: ./backend
    ports:
      - "8000:8000"
    depends_on:
      - mqtt

6. เครือข่ายภายนอก: Cloudflare Tunnel & Tailscale

Cloudflare Tunnel (เปิดเว็บให้คนภายนอกเข้าถึง):

# ติดตั้ง cloudflared (Linux ARM สำหรับ Raspberry Pi)
curl -L --output cloudflared.deb \
  https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-arm.deb
sudo dpkg -i cloudflared.deb

# ล็อกอินและสร้างอุโมงค์
cloudflared tunnel login
cloudflared tunnel create my-iot-hub
cloudflared tunnel run my-iot-hub

Tailscale VPN (รีโมท SSH จากนอกบ้าน):

# ติดตั้ง Tailscale บน Raspberry Pi
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up

# ดู IP ส่วนตัว (Tailscale)
tailscale ip -4

# SSH จากเครื่องอื่น
ssh pi@100.x.y.z

7. เฟิร์มแวร์ ESP32 / PlatformIO

platformio.ini:

[env:esp32dev]
platform = espressif32
board = esp32dev
framework = arduino
monitor_speed = 115200
lib_deps =
    knolleary/PubSubClient @ ^2.8
    bblanchon/ArduinoJson @ ^6.21.3

Non-blocking MQTT Reconnect (ใช้ millis):

unsigned long lastReconnectAttempt = 0;

void loop() {
    if (!client.connected()) {
        unsigned long now = millis();
        if (now - lastReconnectAttempt > 5000) {
            lastReconnectAttempt = now;
            if (client.connect("ESP32_Node")) {
                client.subscribe("actuators/led");
            }
        }
    } else {
        client.loop();
    }
}

หลีกเลี่ยง Heap Fragmentation ด้วย StaticJsonDocument:

void sendTelemetry(float temp) {
    // จองหน่วยความจำใน Stack (ปลอดภัยกว่า DynamicJsonDocument)
    StaticJsonDocument<200> doc;
    doc["sensor_name"] = "esp32_temp";
    doc["value"] = temp;

    char buffer[256];
    serializeJson(doc, buffer);
    client.publish("sensors/data", buffer);
}

XSS Prevention ฝั่ง Dashboard JS:

// ❌ อันตราย — อย่าใช้
element.innerHTML = data.sensor_name;

// ✅ ปลอดภัย — ใช้ textContent เสมอ
element.textContent = data.sensor_name;
